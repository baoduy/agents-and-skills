# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Purpose

Personal collection of Claude Code **skills** and **agents** authored as Markdown, published as the npm package `@drunkcoding/agents-and-skills` and installable via [`npx skills`](https://github.com/vercel-labs/skills). No runtime source code, no build system, no tests. Edits are almost always content edits to `.md` files.

## Layout

- `.claude/skills/<namespace>/<skill>/SKILL.md` — skill definitions loaded by Claude Code and discovered by `npx skills add`. Currently only the `gitnexus/` namespace: `gitnexus-exploring`, `gitnexus-impact-analysis`, `gitnexus-debugging`, `gitnexus-refactoring`, `gitnexus-guide`, `gitnexus-cli`.
- `.claude-plugin/plugin.json` — plugin manifest listing every published skill path. **Update this list when adding/removing a skill** so marketplace consumers see the change.
- `package.json` — npm metadata for `@drunkcoding/agents-and-skills`. Bump `version` together with `.claude-plugin/plugin.json` `version` before `npm publish --access public`.
- `.npmignore` — excludes `.git`, `.gitnexus/`, logs from the published tarball. `files` whitelist in `package.json` takes precedence.
- `Karpathy-CLAUDE.md` — behavioral baseline (think-before-code, surgical diffs, stay in scope). Treat as house style for any code work later added here.
- `AGENTS.md` — mirror of `CLAUDE.md` for non-Claude agent runtimes. **Keep in sync** when editing `CLAUDE.md`. The GitNexus block in both files is auto-regenerated by `npx gitnexus analyze`; do not hand-edit between the `<!-- gitnexus:start -->` / `<!-- gitnexus:end -->` markers.
- `.gitnexus/` — index artifacts (gitignored content; `meta.json` tracked).

## Working in this repo

- Edits are almost always to a single `SKILL.md`. Read the whole file before changing — skill frontmatter (`name`, `description`) drives when Claude loads the skill **and** when `npx skills` indexes it.
- Adding a new skill: create `.claude/skills/<namespace>/<skill>/SKILL.md` with valid `name` + `description` frontmatter, then add its path to the `skills` array in `.claude-plugin/plugin.json`, then bump version in `package.json` + `plugin.json`.
- No tests. Run `npx gitnexus analyze` only if the GitNexus block needs refresh after large structural changes. Run `npm publish --access public` only when explicitly asked.
- Follow `Karpathy-CLAUDE.md`: minimal diff, no scope creep, preserve existing prose unless the task requires changes.

# GitNexus — Code Intelligence

This project is indexed by GitNexus as **agents-and-skills** (13 symbols, 11 relationships, 0 execution flows). Use the GitNexus MCP tools to understand code, assess impact, and navigate safely.

> If any GitNexus tool warns the index is stale, run `npx gitnexus analyze` in terminal first.

## Always Do

- **MUST run impact analysis before editing any symbol.** Before modifying a function, class, or method, run `gitnexus_impact({target: "symbolName", direction: "upstream"})` and report the blast radius (direct callers, affected processes, risk level) to the user.
- **MUST run `gitnexus_detect_changes()` before committing** to verify your changes only affect expected symbols and execution flows.
- **MUST warn the user** if impact analysis returns HIGH or CRITICAL risk before proceeding with edits.
- When exploring unfamiliar code, use `gitnexus_query({query: "concept"})` to find execution flows instead of grepping. It returns process-grouped results ranked by relevance.
- When you need full context on a specific symbol — callers, callees, which execution flows it participates in — use `gitnexus_context({name: "symbolName"})`.

## Never Do

- NEVER edit a function, class, or method without first running `gitnexus_impact` on it.
- NEVER ignore HIGH or CRITICAL risk warnings from impact analysis.
- NEVER rename symbols with find-and-replace — use `gitnexus_rename` which understands the call graph.
- NEVER commit changes without running `gitnexus_detect_changes()` to check affected scope.
